# 九、可观测性与追踪

> **核心文件**：`_instrumentation.py`、`_otel_messages.py`、`models/instrumented.py`

---

## 9.1 设计目标

pydantic-ai 原生支持 [OpenTelemetry](https://opentelemetry.io/) 规范，并深度集成 [Logfire](https://logfire.pydantic.dev/)（Pydantic 官方的 LLM 可观测平台）。

```
pydantic-ai Agent 运行
│
├── OTel Tracer (Span 层级)
│   ├── agent run                      ← 整个 Agent 运行
│   │   ├── model request              ← 每次模型调用
│   │   ├── running tool <name>        ← 工具执行
│   │   └── running output function    ← 输出验证
│
└── OTel Logger (Event 层级，v4+)
    ├── gen_ai.system.message          ← 系统消息
    ├── gen_ai.user.message            ← 用户消息
    ├── gen_ai.assistant.message       ← 模型回复
    └── gen_ai.tool.message            ← 工具结果
```

## 9.2 启用追踪的三种方式

### 方式一：全局 instrument_all()

```python
from pydantic_ai.agent import Agent
import logfire

logfire.configure()
Agent.instrument_all()  # 对所有 Agent 实例生效
```

### 方式二：单个 Agent

```python
agent = Agent('openai:gpt-5.2', instrument=True)  # 使用默认设置
agent = Agent('openai:gpt-5.2', instrument=InstrumentationSettings(include_content=False))
```

### 方式三：包装模型

```python
from pydantic_ai.models.instrumented import InstrumentedModel, InstrumentationSettings

model = InstrumentedModel(
    'openai:gpt-5.2',
    InstrumentationSettings(include_content=True)
)
```

## 9.3 InstrumentationSettings — 追踪配置

**文件**：`models/instrumented.py`

```python
@dataclass
class InstrumentationSettings:
    tracer: Tracer               # OTel Tracer 实例
    logger: Logger               # OTel Logger 实例（用于 'logs' event_mode）
    event_mode: Literal['attributes', 'logs'] = 'attributes'
    # 'attributes': 消息内容作为 Span 属性记录
    # 'logs':       消息内容作为 OTel Log Records 记录（推荐，更结构化）

    include_binary_content: bool = True   # 是否记录二进制内容（图像等）
    include_content: bool = True           # 是否记录消息文本（隐私敏感时设为 False）
    version: Literal[1, 2, 3, 4, 5] = DEFAULT_INSTRUMENTATION_VERSION
    use_aggregated_usage_attribute_names: bool = False
```

**`include_content=False`** 的典型场景：合规要求（GDPR、HIPAA），不记录用户数据，但保留 Token 用量和结构信息。

### Span 属性（Gen AI 语义约定）

模型请求 Span 的标准属性：

```
gen_ai.system                    提供商（'openai', 'anthropic'）
gen_ai.request.model             模型名称（'gpt-5.2'）
gen_ai.request.temperature       温度参数
gen_ai.request.max_tokens        最大 token 数
gen_ai.request.top_p             top-p 参数
gen_ai.usage.input_tokens        输入 token 数
gen_ai.usage.output_tokens       输出 token 数
gen_ai.usage.details.cache_read_tokens  缓存命中 token
gen_ai.agent.name               Agent 名称（从 OTel baggage 读取）
gen_ai.agent.call.id            Agent 运行 ID
```

## 9.5 消息 Span 记录（OTel Events）

**文件**：`_otel_messages.py`

按照 [OTel GenAI 语义约定](https://opentelemetry.io/docs/specs/semconv/gen-ai/)，各类消息部分映射为 OTel 事件：

### v2/v3 版本（'attributes' event_mode）

消息内容序列化为 JSON 字符串，记录在 Span 属性中：
- `gen_ai.prompt` → 输入消息列表
- `gen_ai.completion` → 输出消息列表

### v4+ 版本（'logs' event_mode，推荐）

每条消息作为独立的 OTel Log Record：

```python
class TextPart(TypedDict):
    type: Literal['text']
    content: NotRequired[str]   # include_content=False 时省略

class ToolCallPart(TypedDict):
    type: Literal['tool_call']
    id: str
    name: str
    arguments: NotRequired[JsonValue]

class ToolCallResponsePart(TypedDict):
    type: Literal['tool_call_response']
    id: str
    name: str
    result: NotRequired[JsonValue]
```

Log Record 的 `event.name` 属性：
- 系统消息：`gen_ai.system.message`
- 用户消息：`gen_ai.user.message`
- 助手回复：`gen_ai.assistant.message`
- 工具结果：`gen_ai.tool.message`

## 9.6 InstrumentationNames — 版本化的 Span 命名

**文件**：`_instrumentation.py`

不同版本的 Span 名称不同：

| 版本 | Agent Span | Tool Span | 工具参数属性 |
|------|-----------|-----------|------------|
| v1/v2 | `agent run` | `running tool` | `tool_arguments` |
| v3+ | `invoke_agent {name}` | `execute_tool {name}` | `gen_ai.tool.call.arguments` |

```python
@classmethod
def for_version(cls, version: int) -> Self:
    if version <= 2:
        return cls(
            agent_run_span_name='agent run',
            tool_span_name='running tool',
            tool_arguments_attr='tool_arguments',
            ...
        )
    else:
        return cls(
            agent_run_span_name='invoke_agent',  # 格式化时加 agent_name
            tool_span_name='execute_tool',        # 格式化时加 tool_name
            tool_arguments_attr='gen_ai.tool.call.arguments',
            ...
        )
```

## 9.7 OTel Baggage — 跨 Span 传递 Agent 信息

Agent 运行时通过 OTel Baggage 传递 Agent 名称和 Run ID：

```python
AGENT_NAME_BAGGAGE_KEY = 'gen_ai.agent.name'
RUN_ID_BAGGAGE_KEY = 'gen_ai.agent.call.id'

def get_agent_run_baggage_attributes() -> dict[str, Any]:
    """读取 OTel baggage 中的 Agent 信息，作为当前 Span 属性。"""
```

这允许模型请求 Span（通常在嵌套上下文中创建）自动继承 Agent 名称，而无需显式传递。

## 9.8 Logfire 集成

pydantic-ai 与 [Logfire](https://logfire.pydantic.dev/) 深度集成：

- Logfire 实现了 OTel API，`Agent(instrument=True)` 在 Logfire 上下文中会自动使用 Logfire 的追踪
- pydantic_graph 也通过 `logfire_span` 创建 Span（在 `_utils.py` 中）
- Temporal 集成有专用的 `_logfire.py`，处理 Activity 与 Logfire 的交互

### Durable Execution 与 Logfire

```python
# pydantic_graph/_utils.py
def logfire_span(name: str, **kwargs) -> AbstractContextManager:
    """创建 logfire Span（如果 logfire 可用），否则使用 NoOp。"""
    try:
        import logfire
        return logfire.span(name, **kwargs)
    except ImportError:
        return nullcontext()
```

## 9.9 追踪的 Token 成本计算

`ModelResponse` 存储了完整的 `RequestUsage`，`InstrumentedModel` 在模型请求结束后通过 `genai_prices` 库计算费用：

```python
from genai_prices import calc_price

price = calc_price(
    model_id=f"{model.system}:{model.model_name}",
    input_tokens=usage.input_tokens,
    output_tokens=usage.output_tokens,
    cache_read_tokens=usage.cache_read_tokens,
)
# 结果记录为 Span 属性 'gen_ai.usage.cost'
```

## 总结

```
关键设计决策
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 无侵入包装（InstrumentedModel）：
   任何 Model 实现都可以在不修改代码的情况下
   获得 OTel 追踪支持

2. 版本化语义（InstrumentationSettings.version）：
   随着 OTel GenAI 规范的演进，新版本支持更标准的属性名
   旧代码可以通过固定 version=2 保持兼容

3. 内容可选记录（include_content）：
   隐私敏感场景可禁用内容记录，只保留结构和指标
   解耦"能否追踪"和"是否记录内容"

4. Baggage 传播：
   Agent 名称和 Run ID 通过 OTel Baggage 跨 Span 传播
   嵌套的模型请求 Span 自动继承 Agent 上下文

5. OTel 标准先行：
   遵循 OTel Gen AI 语义约定（gen_ai.*）
   保证与 OpenTelemetry 生态（Jaeger、Tempo 等）的兼容性
```
