## 14.1 系统提示词 vs 指令的区别

pydantic-ai 区分两种"给 Agent 的文本指导"：

| 概念 | 字段 | 作用 |
|------|------|------|
| **system_prompt** | `Agent._system_prompts` | 每次运行时注入到模型的系统消息，与历史对话一起发送 |
| **instructions** | `Agent._instructions` | 运行时的"活动指令"，每次对话可以不同，与历史消息分离 |

两者都支持：静态字符串、动态函数（取 `RunContext`）、`TemplateStr`（Handlebars 模板）。

## 14.2 SystemPromptRunner — 系统提示词执行器

**文件**：`_system_prompt.py`

```python
@dataclass
class SystemPromptRunner(Generic[AgentDepsT]):
    """封装一个系统提示词函数，处理同步/异步和是否接受 ctx 参数。"""

    function: SystemPromptFunc[AgentDepsT]
    dynamic: bool = False   # True: 每次运行步骤都重新评估（用于变化的上下文）

    async def run(self, run_context: RunContext[AgentDepsT]) -> str | None:
        """执行函数，返回字符串或 None（None 表示跳过此提示词）。"""
```

### 注册方式

```python
agent = Agent('openai:gpt-5.2')

# 方式一：静态字符串
agent = Agent('openai:gpt-5.2', system_prompt='You are a helpful assistant.')

# 方式二：装饰器函数（不接受 ctx）
@agent.system_prompt
def my_prompt() -> str:
    return f'Today is {date.today()}'

# 方式三：装饰器函数（接受 ctx，可访问依赖）
@agent.system_prompt
async def context_prompt(ctx: RunContext[MyDeps]) -> str:
    user = await ctx.deps.get_user()
    return f'User: {user.name}'

# 方式四：动态提示词（每个步骤都重新评估）
@agent.system_prompt(dynamic=True)
def step_aware_prompt() -> str:
    return f'Current time: {datetime.now()}'
```

## 14.4 TemplateStr — Handlebars 模板字符串

**文件**：`_template.py`

```python
class TemplateStr(Generic[AgentDepsT]):
    """使用 Handlebars 语法的动态模板字符串，渲染 RunContext.deps。"""
```

### 使用示例

```python
from dataclasses import dataclass
from pydantic_ai import Agent, TemplateStr

@dataclass
class MyDeps:
    name: str
    language: str

agent = Agent(
    'openai:gpt-5.2',
    deps_type=MyDeps,
    instructions=TemplateStr(
        'You are a helpful assistant for {{name}}. '
        'Always respond in {{language}}.'
    ),
)

# 运行时，deps 被注入到模板中
result = await agent.run('Hello', deps=MyDeps(name='Alice', language='Chinese'))
```

### Handlebars 模板语法

- `{{field_name}}`：插入字段值
- `{{#if condition}}...{{/if}}`：条件块
- `{{#each list}}{{this}}{{/each}}`：循环块

### Pydantic 验证集成

`TemplateStr` 实现了 `__get_pydantic_core_schema__`，通过 Pydantic 的验证系统自动识别：

```python
# 在 Pydantic 验证时：
# - 包含 '{{' 的字符串 → 编译为 TemplateStr
# - 不含 '{{' 的字符串 → 回落到普通 str 分支

# 在 Dataset/Spec 的 YAML 文件中：
# instructions: "Hello {{name}}"  → 自动解析为 TemplateStr
```

### 验证上下文传递


## 14.5 format_as_xml — XML 提示词格式化

**文件**：`format_prompt.py`

```python
def format_as_xml(
    obj: Any,
    root_tag: str | None = None,
    item_tag: str = 'item',
    none_str: str = 'null',
    indent: str | None = '  ',
    include_field_info: bool = False,
) -> str:
```

### 典型用途

LLM 对 XML 格式的结构化数据理解效果通常优于 JSON：

```python
from pydantic_ai import format_as_xml
from dataclasses import dataclass

@dataclass
class User:
    name: str
    age: int

print(format_as_xml(User(name='Alice', age=30), root_tag='user'))
# <user>
#   <name>Alice</name>
#   <age>30</age>
# </user>

print(format_as_xml({'items': [1, 2, 3]}, root_tag='data'))
# <data>
#   <items>
#     <item>1</item>
#     <item>2</item>
#     <item>3</item>
#   </items>
# </data>
```

### 支持的类型

| 类型 | 示例 |
|------|------|
| `str`, `int`, `float` | 直接输出文本 |
| `bool` | `true`/`false` |
| `date`, `datetime`, `time` | ISO 8601 格式 |
| `timedelta` | `HH:MM:SS` 格式 |
| `UUID`, `Decimal` | 字符串形式 |
| `Enum` | 枚举值 |
| `None` | 配置的 `none_str`（默认 `null`） |
| `Mapping`（dict 等）| 键作为子标签 |
| `Iterable`（list 等）| 用 `item_tag` 包裹 |
| `dataclass`, `BaseModel` | 字段作为子标签 |

## 14.6 NamedSpec — YAML/JSON 对象规格说明

**文件**：`_spec.py`

`NamedSpec` 是一个通用的序列化/反序列化机制，允许在 YAML/JSON 中以紧凑方式表示 Python 对象：

```python
class NamedSpec(BaseModel):
    """从序列化参数构造命名对象的规格说明。"""

    name: str                                  # 类名
    arguments: None | tuple[Any] | dict[str, Any]  # 构造参数
```

### 序列化格式

```yaml
# 格式1：无参数
- EqualsExpected

# 格式2：单个位置参数
- Equals: 42

# 格式3：关键字参数
- MaxDuration:
    max_duration: 1.0
    
# 格式4：关键字参数（多个）
- LLMJudge:
    rubric: "Output is correct"
    model: openai:gpt-5.2
```

对应 Python：
```python
EqualsExpected()
Equals(42)
MaxDuration(max_duration=1.0)
LLMJudge(rubric='Output is correct', model='openai:gpt-5.2')
```

## 14.7 `@agent.system_prompt` 装饰器的完整流程

```
1. 用户调用 @agent.system_prompt 装饰器
         │
2. 装饰函数被包装为 SystemPromptRunner
         │
3. 注册到 agent._system_prompt_functions 列表
         │
4. Agent 开始运行时（在 UserPromptNode 中）
         │
5. 遍历 _system_prompt_functions，对每个 runner 调用 runner.run(run_context)
         │
6. 返回值为字符串 → 追加到 SystemPromptPart
   返回值为 None → 跳过（允许条件性提示词）
         │
7. 所有提示词合并到 ModelRequest
```

### dynamic=True 的行为

```python
@agent.system_prompt(dynamic=True)
def refresh_prompt() -> str:
    return f'Current state: {get_current_state()}'
```

- `dynamic=False`（默认）：每次 Agent **运行**时评估一次
- `dynamic=True`：每次 Agent **运行步骤**时重新评估（包括工具调用后的下一步）

## 14.8 指令与系统提示词的消息构建对比

```
system_prompt（系统提示词）：
  → 注入为 ModelRequest 的 SystemPromptPart
  → 随完整消息历史传递
  → 模型在每轮对话都能看到

instructions（指令）：
  → 也注入为 SystemPromptPart，但位于独立的消息构建阶段
  → 设计上与历史消息分离
  → 适合描述"本次运行"的上下文，而非持久的角色设定
```

## 总结

```
指令与模板体系设计
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 静态/动态统一接口：
   str（静态）、函数（动态）、TemplateStr（模板）
   都可以作为 system_prompt 或 instructions 传入
   SystemPromptRunner 统一处理同步/异步/有无 ctx

2. TemplateStr 的 Pydantic 集成：
   含 '{{' 的字符串在 Pydantic 验证时自动升级为 TemplateStr
   YAML 配置文件中无需显式声明模板类型

3. Handlebars 作为模板语言：
   通过 pydantic-handlebars 库（可选依赖）支持
   deps 对象通过 TypeAdapter 序列化后传入模板

4. format_as_xml 补充提示词构建：
   XML 格式的结构化数据比 JSON 对 LLM 更友好
   支持所有常见 Python 类型，包括 Pydantic 模型和 dataclass

5. NamedSpec 的 YAML 序列化：
   评估器、工具等对象可以在 YAML 中以紧凑格式声明
   注册表机制（name → class）支持可扩展的自定义类型
```
