## 11.1 设计目标

pydantic-ai 内置了两种主流前端流式协议的适配器：

```
pydantic-ai Agent 运行（内部事件流）
         │
         ▼
  NativeEvent（AgentStreamEvent | AgentRunResultEvent）
         │
    UIEventStream（事件转换层）
    ┌────┴────────────────────┐
    ▼                         ▼
AGUIEventStream         VercelAIEventStream
（AG-UI 协议）         （Vercel AI SDK 协议）
    │                         │
    ▼                         ▼
SSE 流（HTTP Streaming）  SSE 流（HTTP Streaming）
    │                         │
    ▼                         ▼
前端框架（ag-ui-kit）    前端框架（Next.js / useChat）
```

## 11.2 核心抽象：UIEventStream

**文件**：`ui/_event_stream.py`

```python
@dataclass
class UIEventStream(ABC, Generic[RunInputT, EventT, AgentDepsT, OutputDataT]):
    """将 pydantic-ai 内部事件转换为协议特定事件的基类。"""

    run_input: RunInputT
    accept: str | None = None      # Accept 请求头，决定编码格式
    message_id: str = field(...)   # 消息 ID（UUID）

    @abstractmethod
    async def _transform(
        self,
        events: AsyncIterator[NativeEvent],
    ) -> AsyncIterator[EventT]:
        """将 NativeEvent 转换为协议特定的 EventT。"""
```

### 内部事件类型（NativeEvent）

```python
NativeEvent = AgentStreamEvent | AgentRunResultEvent[Any]
```

`AgentStreamEvent` 包含：
- `PartStartEvent` / `PartDeltaEvent` / `PartEndEvent`：文本、工具调用的开始/增量/结束
- `FunctionToolCallEvent` / `FunctionToolResultEvent`：函数工具调用与结果
- `FinalResultEvent`：最终输出完成事件

## 11.3 AG-UI 协议适配器

**文件**：`ui/ag_ui/`

[AG-UI（Agent-User Interaction Protocol）](https://docs.ag-ui.com) 是一个开放的 Agent ↔ 前端通信协议。

### 事件映射

| pydantic-ai 内部事件 | AG-UI 事件 |
|---------------------|-----------|
| `PartStartEvent(TextPart)` | `TextMessageStartEvent` |
| `PartDeltaEvent(TextPartDelta)` | `TextMessageContentEvent` |
| `PartEndEvent(TextPart)` | `TextMessageEndEvent` |
| `PartStartEvent(ThinkingPart)` | `ThinkingStartEvent` |
| `PartDeltaEvent(ThinkingPartDelta)` | `ThinkingTextMessageContentEvent` |
| `PartStartEvent(ToolCallPart)` | `ToolCallStartEvent` |
| `PartDeltaEvent(ToolCallPartDelta)` | `ToolCallArgsEvent` |
| `FunctionToolResultEvent` | `ToolCallResultEvent` |
| Run 开始/结束 | `RunStartedEvent` / `RunFinishedEvent` |

### AGUIAdapter — HTTP 请求处理

```python
class AGUIAdapter(UIAdapter[RunAgentInput, BaseEvent, AgentDepsT, OutputDataT]):
    """AG-UI HTTP 适配器，处理 Starlette/FastAPI 请求。"""

    @classmethod
    async def dispatch_request(
        cls,
        request: Request,          # Starlette/FastAPI 请求
        agent: AbstractAgent,
        deps: AgentDepsT = None,
        ...
    ) -> Response:                 # SSE 流式响应
```

### AGUIApp — 完整 FastAPI 应用

```python
class AGUIApp:
    """创建完整的 AG-UI FastAPI 应用，包含 /run 端点。"""

    def __init__(self, agent: AbstractAgent, ...):
        ...

    def asgi(self) -> ASGIApp:
        """返回 ASGI 应用实例。"""
```

## 11.5 UIAdapter — 请求处理基类

**文件**：`ui/_adapter.py`

```python
class UIAdapter(ABC, Generic[RunInputT, EventT, AgentDepsT, OutputDataT]):
    """HTTP 请求处理基类，包含状态管理和认证逻辑。"""

    agent: ClassVar[AbstractAgent]

    @classmethod
    @abstractmethod
    async def dispatch_request(
        cls,
        request: Request,
        **kwargs,
    ) -> Response:
        """处理 HTTP 请求，返回 SSE 流式响应。"""
```

### StateHandler — 状态管理协议

```python
@runtime_checkable
class StateHandler(Protocol):
    """状态管理协议：要求是 dataclass 且有 `state` 字段。"""
    __dataclass_fields__: ClassVar[dict]

    @property
    def state(self) -> Any: ...

    @state.setter
    def state(self, state: Any) -> None: ...

StateDeps = TypeVar('StateDeps', bound=StateHandler)
```

状态在多轮对话中跨请求持久化，通过 `StateDeps` 注入 Agent 依赖。

## 11.7 OnCompleteFunc — 运行完成回调

```python
OnCompleteFunc = (
    Callable[[AgentRunResult], None]          # 同步回调
    | Callable[[AgentRunResult], Awaitable[None]]  # 异步回调
    | Callable[[AgentRunResult], AsyncIterator[EventT]]  # 产生额外事件
)
```

允许在 Agent 运行完成后：
- 记录消息历史到数据库
- 触发后续工作流
- 向前端发送自定义最终事件

## 11.8 SSE 编码格式

`SSE_CONTENT_TYPE = 'text/event-stream'`

标准 SSE 格式：
```
data: {"type":"text-start","id":"abc123"}

data: {"type":"text-delta","delta":"Hello","id":"abc123"}

data: {"type":"finish","finishReason":"stop"}

```

不同协议的编码差异通过 `accept` 请求头协商。

## 11.9 format_as_xml — 提示词格式化工具

**文件**：`format_prompt.py`

```python
def format_as_xml(
```

## 11.10 集成示例

### FastAPI + AG-UI

```python
from fastapi import FastAPI
from pydantic_ai import Agent
from pydantic_ai.ui.ag_ui import AGUIAdapter

agent = Agent('openai:gpt-5.2')

class MyAdapter(AGUIAdapter):
    agent = agent

app = FastAPI()
app.add_route('/run', MyAdapter.dispatch_request, methods=['POST'])
```

### Next.js + Vercel AI SDK

```python
from fastapi import Request
from pydantic_ai.ui.vercel_ai import VercelAIAdapter
from pydantic_ai import Agent

agent = Agent('openai:gpt-5.2')

class MyAdapter(VercelAIAdapter):
    agent = agent

@app.post('/api/chat')
async def chat(request: Request):
    return await MyAdapter.dispatch_request(request)
```

## 总结

```
UI 层设计模式
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 协议无关的内部事件流：
   NativeEvent 作为内部标准，通过 UIEventStream 派生适配
   支持同时集成多种前端协议，不修改 Agent 核心逻辑

2. SSE 作为传输层：
   所有协议都基于 text/event-stream（SSE）
   天然支持流式传输，前端可边生成边显示

3. StateHandler 协议：
   通过 dataclass + state 属性实现轻量级状态管理
   跨请求状态持久化不依赖具体框架

4. Starlette/FastAPI 集成：
   UIAdapter 基于 Starlette Request/Response
   兼容所有 ASGI 框架，包括 FastAPI、Litestar

5. OnCompleteFunc 扩展点：
   运行完成回调支持同步/异步/异步生成器三种形式
   允许在不修改事件流的情况下添加后处理逻辑
```
