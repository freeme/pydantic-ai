## 12.1 设计目标

pydantic-ai 内置独立的向量嵌入系统，与 LLM 模型系统设计一致：

```
Embedder（高级接口）
    │
    ▼
EmbeddingModel（抽象基类）
    │
    ├── OpenAIEmbeddingModel       ('openai:text-embedding-3-small')
    ├── CohereEmbeddingModel       ('cohere:embed-v4.0')
    ├── GoogleEmbeddingModel       ('google-gla:gemini-embedding-001')
    ├── BedrockEmbeddingModel      ('bedrock:amazon.titan-embed-text-v2:0')
    ├── VoyageAIEmbeddingModel     ('voyageai:voyage-4-large')
    ├── SentenceTransformerEmbeddingModel  ('sentence-transformers:...')  ← 本地运行
    ├── InstrumentedEmbeddingModel         ← OTel 追踪包装
    └── WrapperEmbeddingModel              ← 自定义包装基类
```

## 12.2 Embedder — 高级接口

```python
@dataclass
class Embedder:
    """向量嵌入的高级接口，对标 Agent 的设计风格。"""

    instrument: InstrumentationSettings | bool | None

    def __init__(
        self,
        model: EmbeddingModel | KnownEmbeddingModelName | str,
        *,
        settings: EmbeddingSettings | None = None,
        defer_model_check: bool = True,
        instrument: InstrumentationSettings | bool | None = None,
    )
```

### 核心方法


## 12.3 EmbeddingModel ABC

**文件**：`embeddings/base.py`

```python
class EmbeddingModel(ABC):
    """嵌入模型抽象基类。"""

    @property
    @abstractmethod
    def model_name(self) -> str:
        """模型名称（如 'text-embedding-3-small'）。"""

    @property
    @abstractmethod
    def system(self) -> str:
        """提供商标识（如 'openai', 'cohere'）。"""

    @abstractmethod
    async def embed(
        self,
        inputs: str | Sequence[str],
        *,
        input_type: EmbedInputType,
        settings: EmbeddingSettings | None = None,
    ) -> EmbeddingResult:
        """生成向量嵌入。"""

    async def max_input_tokens(self) -> int | None:
        """最大输入 token 数（None 表示未知）。"""

    async def count_tokens(self, text: str) -> int:
        """Token 计数（默认抛出 NotImplementedError）。"""
```

## 12.4 EmbedInputType — 查询 vs 文档

```python
EmbedInputType = Literal['query', 'document']
```

许多嵌入模型（如 Cohere、VoyageAI）会根据输入类型优化向量：
- `'query'`：语义检索的查询，更关注"语义匹配"
- `'document'`：存储的文档，更关注"内容表示"

## 12.5 EmbeddingResult — 嵌入结果

```python
@dataclass
class EmbeddingResult:
    embeddings: Sequence[Sequence[float]]  # 向量列表，一个输入对应一个向量
    inputs: Sequence[str]                  # 原始输入文本列表
    input_type: EmbedInputType            # 查询或文档类型
    model_name: str                       # 模型名称
    provider_name: str                    # 提供商名称
    timestamp: datetime                   # 请求时间
    usage: RequestUsage                   # Token 用量统计
    provider_details: dict | None         # 提供商特定数据

    def __getitem__(self, input_text: str) -> Sequence[float]:
        """通过输入文本访问对应的向量（索引查找）。"""
```

## 12.6 EmbeddingSettings — 通用配置

```python
class EmbeddingSettings(TypedDict, total=False):
    dimensions: int    # 向量维度（支持降维）
    truncate: bool     # 是否截断超长输入（默认 False）
```

各提供商扩展了额外的特定配置（通过前缀区分）：

| 提供商 | 特定设置类 | 额外字段 |
|--------|-----------|---------|
| OpenAI | `OpenAIEmbeddingSettings` | `openai_user` |
| Cohere | `CohereEmbeddingSettings` | `cohere_input_type`、`cohere_truncate` |
| Sentence Transformers | `SentenceTransformersEmbeddingSettings` | `sentence_transformers_device`、`normalize_embeddings`、`batch_size` |

## 12.7 惰性加载的 infer_embedding_model()

```python
def infer_embedding_model(model: EmbeddingModel | str) -> EmbeddingModel:
    """根据 'provider:model-name' 格式的字符串推断并实例化嵌入模型。"""

    provider_name, model_name = model.split(':', maxsplit=1)
    provider = infer_provider(provider_name)

    if provider_name == 'openai':
        return OpenAIEmbeddingModel(model_name, provider=provider)
    elif provider_name == 'cohere':
        return CohereEmbeddingModel(model_name, provider=provider)
    elif provider_name == 'bedrock':
        return BedrockEmbeddingModel(model_name, provider=provider)
    elif provider_name in ('google-gla', 'google-vertex'):
        return GoogleEmbeddingModel(model_name, provider=provider)
    elif provider_name == 'sentence-transformers':
        return SentenceTransformerEmbeddingModel(model_name)  # 无需 provider
    elif provider_name == 'voyageai':
        return VoyageAIEmbeddingModel(model_name, provider=provider)
```

## 12.10 WrapperEmbeddingModel — 透明代理基类

```python
@dataclass
class WrapperEmbeddingModel(EmbeddingModel):
    """可扩展的嵌入模型包装基类，默认代理所有方法。"""

    wrapped: EmbeddingModel

    async def embed(self, inputs, *, input_type, settings=None) -> EmbeddingResult:
        return await self.wrapped.embed(inputs, input_type=input_type, settings=settings)

    async def max_input_tokens(self) -> int | None:
        return await self.wrapped.max_input_tokens()

    async def count_tokens(self, text: str) -> int:
        return await self.wrapped.count_tokens(text)
```

可用于实现：缓存（CachedEmbeddingModel）、速率限制、批处理优化等。

## 总结

```
嵌入模型系统的设计一致性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 与 LLM 系统对称设计：
   Embedder ≈ Agent（高级接口）
   EmbeddingModel ≈ Model（ABC）
   WrapperEmbeddingModel ≈ WrapperModel
   InstrumentedEmbeddingModel ≈ InstrumentedModel

2. query/document 语义区分：
   EmbedInputType 允许提供商针对检索场景优化
   embed_query 和 embed_documents 是 Embedder 的两个主要入口

3. 本地模型的一等公民支持：
   SentenceTransformerEmbeddingModel 与 API 模型统一接口
   不需要 Provider，直接在本地运行

4. 惰性加载（defer_model_check=True）：
   与 Agent 相同的模式，避免导入时检查模型可用性
   第一次调用 embed() 时才实例化底层模型

5. OTel 追踪对称：
   Embedder.instrument_all() 与 Agent.instrument_all() 行为一致
   InstrumentedEmbeddingModel 与 InstrumentedModel 包装模式相同
```
